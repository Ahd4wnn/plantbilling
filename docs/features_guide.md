# Plantora Application Features Guide

This guide details all key features, backend services, and algorithms implemented in the Plantora Billing System.

---

## 1. POS Billing Terminal

The POS Billing Terminal (`/app/bill`) is the core transaction desk of the application. It includes the following components:

### A. Product Grid
- Renders active products in a grid, complete with thumbnail representations, retail pricing, and wholesale index metadata.
- Enables filtering by plant categories (e.g., *indoor plants*, *pots*, *flowering plants*, *fertilisers*).
- Items can be tapped directly to add them to the transaction cart, incrementing quantities automatically.

### B. Barcode Scanner
- Uses the device's native camera stream via a WebRTC video overlay.
- Scans barcodes (typically 1D barcodes like Code 128/EAN-13), decodes the payload, and instantly inserts the matching catalog product into the cashier's active cart.

### C. Quick Add Tool
- Cashiers can handle sales of custom, unlisted, or rare items (e.g., unique hybrid plants) without modifying the catalog.
- Opens a sliding bottom sheet where cashiers input:
  - Custom Product Description.
  - Price per unit.
  - Initial quantity.
- The item is inserted as a custom item in the current transaction cart.

### D. Multi-Option Discount Engine
- Supports two discount formats:
  - **Percentage-based (`percent`)**: Applies a discount dynamically calculated as a percentage of the subtotal (e.g., `10%`).
  - **Flat Amount (`flat`)**: Subtracts a specific monetary value directly from the subtotal (e.g., `Rs. 50`).
- The subtotal, discount reduction, and total due are calculated instantly on the client side using high-precision math libraries (preventing floating-point errors).

### E. Split Payment Settlement
- Accommodates customers paying with cash, digital wallets, or store credit.
- The checkout panel allows splitting the total across:
  - **Cash**: Logged in the cash drawer summary.
  - **UPI**: Generates a dynamic UPI Payment URI and QR code (containing parameters for merchant name, store UPI ID, and exact invoice amount) that customers scan to pay instantly.
  - **Due Balance**: Logged as a credit balance on the customer's ledger, allowing nurseries to track wholesale accounts.

## 2. Advanced Search & Indian English Voice Recognition

To help nurseries find products quickly under dusty or busy greenhouse conditions, Plantora implements a custom voice search system.

### A. Web Audio API Chimes
To provide a premium native feel, the app uses browser oscillators to play low-level audio indicator beeps during voice commands:
- **Initialize**: A dual-rising sine chime (440Hz -> 554.37Hz) signals that the microphone is open.
- **Success**: A rapid three-note chord (523.25Hz -> 659.25Hz -> 783.99Hz) plays when a match is found.
- **Cancel/Fail**: A descending chime (349.23Hz -> 293.66Hz) sounds if search is cancelled or fails.

### B. Indian English Phonetic Normalizer
Speech recognition engines often struggle with regional accents. Plantora includes a custom phonetic mapper in [VoiceSearchButton.tsx](file:///Users/layobaiju/Downloads/bill%20plant%20park/frontend/src/components/VoiceSearchButton.tsx) tailored to common Indian English speech patterns:
1. **Initial Character Corrections**: Words starting with 'x' are mapped to 'z' (e.g., *xenia* becomes *zenia*).
2. **V/W Merger**: Many Indian speakers merge the pronunciations of `V` and `W`. The normalizer converts all instances of `w` to `v` (e.g., *watering* and *vatering* resolve to the same code).
3. **Consonant Simplification**: 
   - `ph` becomes `f`
   - `gh` becomes `g`
   - `kh` becomes `k`
   - `sh` becomes `s` (merges sh and s)
   - `z` becomes `s`
   - `y` becomes `i`
4. **De-duplication**: Double letters are reduced to a single character (e.g., `ll` to `l`, `nn` to `n`).
5. **Vowel Stripping**: All vowels (`a`, `e`, `i`, `o`, `u`) are removed after the first letter, creating a core phonetic signature (e.g., *ficus* -> *fcs*, *hibiscus* -> *hbsc*).

### C. Multi-Algorithmic Match Scoring
When a voice transcript is captured, it is scored against all catalog candidates using five algorithms in sequence:
1. **Exact String Match**: A score of `1.0` is returned for perfect matches.
2. **Substring Inclusion**: Checks if the candidate contains the transcript (e.g., *bonsai* matches *ficus bonsai*).
3. **Jaccard Word Similarity**: Measures the overlap of words in the query and candidate strings.
4. **Phonetic Jaccard Match**: Scores the overlap of the phonetic codes generated by the Indian English Phonetic Normalizer.
5. **Levenshtein Distance**: Measures the edit distance (insertions, deletions, substitutions) between the strings to handle slight typos or misheard characters.

## 3. Cross-Platform Printing Strategy

Plantora uses a decoupled, platform-adaptive printing strategy to support various hardware setups:

- **Android Tablets & Devices**:
  - Connects to local Bluetooth/USB receipt printers via the **RawBT app**.
  - Generates binary ESC/POS commands (adjusted to 58mm or 80mm widths) and encodes them as a Base64 string.
  - Launches the print job using the Android URL intent: `rawbt:base64,<payload>`.
  - Includes a fallback detector that prompts the user to download the RawBT app from the Play Store if the intent fails to launch.
- **Apple iOS Devices (iPad/iPhone)**:
  - Generates a PDF invoice URL and opens it in Safari, allowing the user to print using iOS AirPrint or share the PDF.
- **Desktop (macOS/Windows)**:
  - Creates a hidden, off-screen `<iframe>` dynamically.
  - Injects the styled receipt HTML (A4 invoice format or 58mm/80mm receipt templates).
  - Triggers standard system printing (`window.print()`) inside the iframe context and cleans up the element afterward.

## 4. WhatsApp Cloud API Automated Invoicing

The backend includes a background queuing service to send receipts to customers' phones via WhatsApp:

- **Automatic Delivery**: If enabled by the owner, completing a checkout triggers a background job to send a Meta WhatsApp template message with the invoice details.
- **Optional PDF Generation**: The system can compile a professional A4 PDF invoice, save it to the server's media folder, upload it to the Meta Cloud API, and attach it to the WhatsApp message.
- **Live Status Badges**: The frontend displays real-time badges tracking the message delivery status:
  - `Queued` $\rightarrow$ Added to background worker queue.
  - `Sending` $\rightarrow$ Compiling PDF and uploading to Meta servers.
  - `Sent` $\rightarrow$ Dispatched to the user.
  - `Delivered` $\rightarrow$ Received by the customer's phone.
  - `Read` $\rightarrow$ Opened by the customer (blue ticks).
  - `Failed` $\rightarrow$ Delivery failed (shows error reason).

## 5. Ledger Accounts & Expense Manager

### A. Customer Ledger & Credit Accounts
- Automatically logs customer transactions, contact details, and total checkout volumes.
- Tracks unpaid balances (store credit) in a customer ledger. Shop owners can review credit history and log payments when customers pay off their debt.

### B. Expense Management
- Permits nursery owners to log operational expenses (fertilizer, labor, fuel, packaging, etc.).
- Categorized expense logs are subtracted from gross sales to calculate the nursery's actual daily, weekly, or monthly net profit.
